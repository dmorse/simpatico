279  /*
280  * Implementation Notes:
281  *
282  * The pair list is stored in Atom* pointer arrays atom1Ptrs_, atom2Ptrs_,
283  * and in an integer array first_. Each element of atom1Ptrs_ contains a
284  * pointer to the first atom in a pair (the primary Atom). Each element
285  * of atom2Ptrs_ contains a pointer to the second atom in a pair (the
286  * secondary Atom). Secondary atoms that are neighbors of the same
287  * primary atom are listed consecutively. Element first_[i] contains the
288  * array index of the first element in atom2Ptrs_ that contains a neighbor
289  * of primary atom atom1Ptrs_[i]. Pointers to neighbors of primary atom
290  * *atom1Ptrs_[i] are thus stored in elements first_[i] <= j < first_[i+1]
291  * of atom2Ptrs_. Each pair is included only once, by including only
292  * pairs in which atom2Ptrs_[j]->id() > atom1Ptrs_[i]->id(), so
293  * the Id of the secondary atom in a pair is always greater than that of
294  * the primary atom. The total number of primary atoms in atom1Ptrs_ is
295  * nAtom1_. The total number of secondary atoms in Array atom2Ptrs_ (or
296  * the total number of distinct pairs) is nAtom2_. The total number of
297  * Atoms in the System in nAtom_.
298  *
299  * The only way legal way to loop over all atom pairs, using the public
300  * interface of a PairList, is to use a PairListIterator. See the
301  * documentation of PairListIterator for a discussion of its usage. The
302  * resulting loop over pairs is equivalent to the following double loop
303  * over private member variables of PairList:
304  *
305  * int i, j;
306  * Atom* atom1Ptr;
307  * Atom* atom2Ptr;
308  *
309  * \\ Loop over primary Atoms
310  * for (i = 0; i < nAtom1_; ++i) {
311  * atom1Ptr = atom1Ptrs_[i];
312  *
313  * // Loop over secondary atoms
314  * for (j = first_[i]; j < first_[i+1]; ++j) {
315  * atom2Ptr = atom2Ptrs_[j];
316  *
317  * // ... Do something with Atoms *atom1Ptr and *atom2Ptr.
318  *
319  * }
320  * }
321  *
322  * Note that DArray<int> first_ contains nAtom1_ + 1 elements. The first
323  * element, element first_[0] is equal to 0, and the last element,
324  * first_[nAtom1_], is always equal to nAtom2_, the total number of
325  * pairs.
326  *
327  * The number of primary Atoms nAtom1_ is generally less than nAtom_, the
328  * total number of atoms, because some Atoms have no neighbors with integer
329  * Ids higher than their own. This is always true, for example, for the
330  * Atom with the highest Id. Pointers to such atoms, which do not appear
331  * in the first nAtom1_ elements of atom1Ptrs_, are instead stored in the
332  * last elements of the atom1Ptrs_ array , in elements tList1 + 1, .... ,
333  * atomCapacity_ - 1. The previous position Vectors for these Atoms are
334  * stored in the corresponding elements of the oldPositions_ array.
335  * These pointers and previous positions retained because they are used
336  * in the isCurrent() method to calculate how far these Atoms have moved
337  * since the last time the PairList was built.
338  */
